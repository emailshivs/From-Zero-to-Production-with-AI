# Instructor Guide

## Purpose

The instructor guide helps mentors, teachers, bootcamp leads, and technical reviewers deliver the handbook consistently.

## How to Mentor a Student

Effective mentoring combines structure, patience, and evidence.

Recommended approach:

1. Confirm the learner’s path and goal.
2. Review the dashboard before each session.
3. Ask the learner to explain what they tried.
4. Inspect evidence before giving answers.
5. Guide the learner toward diagnosis.
6. Record blockers and next actions.
7. Encourage reflection after each milestone.

## Mentoring Principles

- Do not complete the learner’s work for them.
- Ask questions before giving solutions.
- Tie feedback to evidence.
- Reinforce professional habits.
- Normalize troubleshooting as part of engineering.

## How to Validate Exercises

Validation should be objective.

Check that:

- Required files exist.
- Commands or actions were run in the correct context.
- Expected outputs match the chapter criteria.
- Screenshots prove the correct state.
- The learner can explain what the evidence means.

## How to Review Screenshots

Review screenshots for:

- Correct filename.
- Correct storage location.
- Relevant visible state.
- Clear crop and readable text.
- No secrets or personal information.
- Caption and alt text when published.

Reject screenshots that are decorative, unreadable, unrelated, or unsafe.

## How to Assess Understanding

Use layered assessment:

1. Recall: can the learner define key terms?
2. Explanation: can the learner describe why the task matters?
3. Application: can the learner complete the work?
4. Validation: can the learner prove it works?
5. Transfer: can the learner apply the idea to a new scenario?

## Feedback Model

Use this feedback format:

| Field | Guidance |
| --- | --- |
| Strength | Identify what was done well |
| Gap | Identify the specific missing evidence or misunderstanding |
| Action | Give the next corrective step |
| Validation | Explain how the learner will prove the fix |

## Instructor Boundaries

Instructors should not mark work complete based only on effort. Completion requires evidence aligned with the chapter, lab, or project criteria.
