# Course Dashboard

## Purpose

The course dashboard defines the learner progress model. It can be implemented as a spreadsheet, learning management system view, GitHub issue board, classroom tracker, or student portfolio checklist.

## Dashboard Fields

| Field | Description | Evidence source |
| --- | --- | --- |
| Learner name | Student or participant name | Enrollment record |
| Learning path | Selected audience path | Learning path selection |
| Completed chapters | Number and percentage of chapters completed | Chapter checklist |
| Completed labs | Number and percentage of labs completed | Lab validation evidence |
| Screenshots submitted | Required screenshots submitted and accepted | Screenshot review |
| Quiz scores | Score per quiz and average score | Quiz answer key |
| Projects completed | Capstone or practical projects finished | Project rubric |
| Overall completion | Weighted completion score | Dashboard calculation |
| Instructor notes | Mentor observations and remediation items | Review comments |

## Completion Calculation

Use a weighted completion model:

| Area | Weight |
| --- | --- |
| Chapters completed | 25% |
| Labs completed | 25% |
| Screenshots accepted | 15% |
| Quiz score average | 15% |
| Projects completed | 20% |

Overall completion is the weighted total of all areas.

## Status Labels

Use consistent progress labels:

| Label | Meaning |
| --- | --- |
| Not started | No evidence submitted |
| In progress | Some evidence submitted |
| Blocked | Learner needs help or environment issue exists |
| Submitted | Evidence is awaiting review |
| Accepted | Evidence meets criteria |
| Needs revision | Evidence does not meet criteria yet |
| Complete | All required evidence is accepted |

## Instructor Use

Instructors should update dashboard status after reviewing evidence. Do not mark a section complete based only on attendance or reading time.

## Learner Use

Learners should use the dashboard to identify what remains, what is blocked, and what evidence must be improved.
