# Progress Tracker

## Purpose

The progress tracker gives learners and instructors a consistent record of completion evidence across chapters, labs, quizzes, screenshots, and projects.

## Tracker Fields

| Field | Description |
| --- | --- |
| Date | Date evidence was completed or reviewed |
| Phase | Handbook phase |
| Chapter | Chapter number or title |
| Lab | Lab identifier, if applicable |
| Evidence type | Chapter, lab, screenshot, quiz, project, reflection |
| Evidence location | Link or file path |
| Status | Not started, in progress, submitted, accepted, needs revision |
| Reviewer | Instructor, peer, or self-review |
| Notes | Blockers, corrections, or next action |

## Weekly Progress Review

At the end of each week, answer:

1. What did I complete?
2. What evidence proves completion?
3. What is blocked?
4. What needs review?
5. What will I complete next week?

## Evidence Quality Rules

Evidence must be:

- Relevant to the task.
- Named clearly.
- Stored where reviewers can find it.
- Free of sensitive information.
- Connected to pass criteria.

## Completion States

| State | Definition |
| --- | --- |
| Not started | No work or evidence yet |
| In progress | Work started but evidence incomplete |
| Submitted | Evidence provided for review |
| Accepted | Evidence meets the criteria |
| Needs revision | Evidence does not meet the criteria |
| Blocked | Learner cannot proceed without help |
