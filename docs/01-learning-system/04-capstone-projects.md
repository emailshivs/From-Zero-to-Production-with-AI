# Capstone Projects

## Purpose

Capstone projects integrate the handbook into portfolio evidence. Each project introduces more responsibility, ambiguity, and professional judgment than the previous project.

## Capstone Progression

| Capstone | Relative complexity | Primary focus |
| --- | --- | --- |
| Student Task Tracker | 1 | Basic product workflow and validation |
| Personal Expense Tracker | 2 | Data, user workflows, and quality checks |
| Notes App | 3 | CRUD behavior, usability, and maintainability |
| Secure SaaS | 4 | Security, validation, and production readiness |
| Wealth Survivor | 5 | Integrated architecture, operations, AI assistance, and business reasoning |

## 1. Student Task Tracker

### Goal

Create a simple task management product concept that supports student organization and evidence-based learning progress.

### Concepts introduced

- Product goal definition.
- Basic user workflow.
- Chapter and lab validation evidence.
- Simple screenshot submission.
- Basic project reflection.

### Required evidence

- Project brief.
- Workflow diagram.
- Validation checklist.
- Accepted screenshots.
- Reflection summary.

### Expected outcome

The learner can explain a simple product, demonstrate completion evidence, and connect features to a user need.

## 2. Personal Expense Tracker

### Goal

Create a product concept that helps an individual track income, expenses, categories, and simple financial decisions.

### Concepts introduced

- Data modeling awareness.
- Input validation.
- Quality checks.
- Privacy considerations.
- Basic reporting.

### Required evidence

- Product brief.
- Data flow diagram.
- Test checklist.
- Screenshot evidence.
- Risk notes.

### Expected outcome

The learner can reason about data, validation, privacy, and user value in a small application.

## 3. Notes App

### Goal

Create a notes application concept that supports creating, reading, updating, deleting, organizing, and searching notes.

### Concepts introduced

- CRUD workflow.
- Information organization.
- Maintainability.
- Regression validation.
- AI-assisted documentation review.

### Required evidence

- Feature map.
- Folder or component diagram.
- Validation results.
- Prompt documentation, if AI is used.
- Quiz or reflection evidence.

### Expected outcome

The learner can connect user workflows to maintainable implementation and validation evidence.

## 4. Secure SaaS

### Goal

Create a SaaS product concept that introduces secure delivery, quality gates, operational readiness, and review evidence.

### Concepts introduced

- Secure configuration.
- Authentication and authorization awareness.
- Quality gates.
- Deployment readiness.
- Security review.
- Incident and rollback thinking.

### Required evidence

- Architecture diagram.
- Threat or risk notes.
- Validation checklist.
- Security review summary.
- Operations readiness checklist.

### Expected outcome

The learner can discuss how a product moves from working software to responsible production readiness.

## 5. Wealth Survivor

### Goal

Create an integrated product concept that helps users survive and improve financially through planning, scenario thinking, learning, and responsible AI assistance.

### Concepts introduced

- Multi-role product design.
- Business objective mapping.
- Risk-aware architecture.
- AI prompt documentation.
- Quality and security evidence.
- Production and governance readiness.

### Required evidence

- Product vision.
- Role-based user journeys.
- Architecture package.
- Skills matrix mapping.
- Validation and screenshot package.
- AI prompt records.
- Final presentation.

### Expected outcome

The learner can synthesize development, quality, security, AI, operations, and architecture practices into a coherent product delivery story.

## Capstone Review Rubric

| Area | Evaluation question |
| --- | --- |
| Business clarity | Does the project solve a clear problem? |
| Technical structure | Are workflows, diagrams, and evidence organized? |
| Validation | Can the learner prove completion? |
| Security | Are obvious risks identified and handled? |
| Communication | Can the learner explain decisions clearly? |
| Portfolio readiness | Is the work presentable to an instructor, interviewer, or stakeholder? |
