# Capstone Progression

## Purpose

This document maps capstones to phases, skills, assessments, and increasing learner independence.

| Capstone | Phase alignment | Concepts introduced | Required evidence | Readiness signal |
| --- | --- | --- | --- | --- |
| Student Task Tracker | Getting Started, Phase 00 | Product framing, simple workflow, evidence capture | Brief, workflow, checklist, screenshots | Learner can explain a small product and evidence. |
| Personal Expense Tracker | Phases 00-02 | Data awareness, validation, quality checks | Data flow, test checklist, screenshot evidence | Learner can validate user-facing behavior. |
| Notes App | Phases 01-04 | Maintainability, CRUD planning, prompt documentation | Feature map, diagrams, prompt records, quiz evidence | Learner can connect implementation to reviewable evidence. |
| Secure SaaS | Phases 02-06 | Quality gates, security review, operations readiness | Architecture, risk notes, operations checklist | Learner can discuss production readiness. |
| Wealth Survivor | All phases | Integrated product, AI, security, operations, enterprise governance | Product vision, journeys, architecture package, skills mapping, presentation | Learner can synthesize the full handbook. |

## Progression Rule

Each capstone should reuse prior evidence and add one new layer of ambiguity, stakeholder reasoning, or professional review.
