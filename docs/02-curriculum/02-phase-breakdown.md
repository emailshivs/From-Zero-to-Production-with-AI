# Phase Breakdown

## Purpose

This document defines the curriculum contract for every phase. Authors should use it before drafting chapter outlines, labs, quizzes, or capstone activities.

## Phase 00: Build First SaaS

| Category | Blueprint |
| --- | --- |
| Purpose | Convert a product idea into an initial working SaaS learning vehicle. |
| Business objective | Prove that a small product can be shaped, built, demonstrated, and validated. |
| Learning objective | Define a SaaS concept, create baseline evidence, and explain a first delivery cycle. |
| Skills gained | Product framing, basic implementation flow, evidence collection. |
| Tools introduced | Editor, Git, GitHub, browser, local runtime. |
| Architecture introduced | Single-app baseline, user journey, simple deployment boundary. |
| Labs | Starter build lab, project brief lab, evidence capture lab. |
| Failure labs | Broken setup lab, missing dependency lab, invalid configuration lab. |
| Wealth Survivor mapping | Introduces the smallest version of Wealth Survivor as a product vision and user need. |
| Enterprise mapping | Maps to MVP discovery, proof-of-concept delivery, and founder technical validation. |
| Deliverables | Project brief, baseline repository, screenshots, validation checklist. |
| Validation evidence | Working state screenshot, command output, checklist, short reflection. |
| Interview readiness | Explain what was built, why it matters, and how completion was validated. |
