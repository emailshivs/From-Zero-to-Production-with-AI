# Course Map

## Purpose

The course map defines the curriculum structure before technical chapters are written. It keeps phases, labs, assessments, tools, architecture, capstones, and enterprise outcomes aligned.

## Curriculum Spine

| Phase | Name | Curriculum role | Primary evidence |
| --- | --- | --- | --- |
| Phase 00 | Build First SaaS | Convert a product idea into an initial working SaaS learning vehicle. | Project brief, baseline repository, screenshots, validation checklist. |
| Phase 01 | Development | Establish professional development workflows and maintainable feature delivery. | Development workflow evidence, feature map, review checklist. |
| Phase 02 | Quality Engineering | Make quality visible through tests, validation, and release confidence. | Quality strategy, test checklist, screenshot evidence, quiz results. |
| Phase 03 | Security Engineering | Introduce security thinking, controls, and review evidence. | Security review, threat notes, control checklist, remediation plan. |
| Phase 04 | AI Engineering | Use AI assistance responsibly with documented prompts and validated output. | Prompt records, reviewed outputs, limitation notes, validation checklist. |
| Phase 05 | Production Operations | Prepare software for deployment, monitoring, troubleshooting, and support. | Operations checklist, deployment evidence, monitoring notes, incident reflection. |
| Phase 06 | Enterprise DevSecOps | Scale delivery practices into governance, automation, and enterprise controls. | Governance map, release checklist, control evidence, architecture review notes. |

## Learning Flow

1. Orient learners to evidence-based completion.
2. Build a small product path.
3. Add professional development workflow.
4. Add quality and validation gates.
5. Add security review and risk thinking.
6. Add responsible AI assistance.
7. Add operations readiness.
8. Add enterprise DevSecOps governance.
9. Demonstrate synthesis through capstones.

## Blueprint Boundaries

This section defines curriculum design only. It does not contain chapter teaching content, implementation walkthroughs, or tool-specific tutorials.
