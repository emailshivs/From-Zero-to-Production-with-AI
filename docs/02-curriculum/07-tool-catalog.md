# Tool Catalogue

## Purpose

The tool catalogue documents every tool category planned for the handbook before technical lessons are written.

| Tool | Purpose | Role in curriculum | Alternatives | Introduced |
| --- | --- | --- | --- | --- |
| MkDocs | Builds the documentation site | Publishing platform | Docusaurus, Sphinx | Repository setup |
| Material for MkDocs | Provides professional documentation UI | Theme and navigation | Plain MkDocs theme | Repository setup |
| Markdown | Stores handbook source | Authoring format | AsciiDoc, reStructuredText | Editorial standards |
| Mermaid | Defines diagrams as text | Diagramming | PlantUML, Draw.io | Diagram standards |
| Git | Tracks changes | Version control | Mercurial | Getting Started |
| GitHub | Hosts repository and PRs | Collaboration | GitLab, Bitbucket | Development |
| IDE or code editor | Edits code and docs | Authoring environment | Any standards-compliant editor | Getting Started |
| Package manager | Installs project dependencies | Dependency workflow | Language-specific alternatives | Development |
| Formatter | Enforces style | Code consistency | Manual formatting | Development |
| Linter | Finds quality issues | Static feedback | Manual review | Development |
| Test runner | Runs automated checks | Validation | Manual testing | Quality Engineering |
| Coverage tool | Shows test coverage signals | Quality evidence | Mutation testing tools | Quality Engineering |
| Browser dev tools | Inspects web behavior | Debugging | Standalone debugging tools | Quality Engineering |
| Secret scanner | Detects committed secrets | Security control | Manual inspection | Security Engineering |
| Dependency scanner | Detects dependency risk | Security control | Vendor scanners | Security Engineering |
| AI assistant | Supports reviewed AI work | AI augmentation | Manual-only workflow | AI Engineering |
| Prompt log | Records prompt purpose and output | AI audit evidence | Issue comments, notebooks | AI Engineering |
| Hosting platform | Runs deployed app or docs | Deployment target | Cloud provider, static host | Production Operations |
| Monitoring dashboard | Shows runtime signals | Operations evidence | Logs only | Production Operations |
| CI/CD platform | Automates checks and releases | DevSecOps workflow | Manual release process | Enterprise DevSecOps |
| Issue tracker | Tracks work and decisions | Planning evidence | Spreadsheet, Kanban board | Enterprise DevSecOps |

## Tool Governance

A tool should appear in chapters only when it supports a learning objective, validation requirement, or capstone deliverable. Tool substitutions must preserve the same evidence outcome.
