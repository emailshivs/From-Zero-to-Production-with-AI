# Enterprise Mappings

## Purpose

Enterprise mappings connect the learning curriculum to professional delivery, governance, security, operations, and architecture practices.

| Phase | Enterprise mapping | Team role alignment | Governance evidence |
| --- | --- | --- | --- |
| Phase 00 | Maps to MVP discovery, proof-of-concept delivery, and founder technical validation. | Product, engineering, QA, security, operations, architecture | Project brief, baseline repository, screenshots, validation checklist.; Working state screenshot, command output, checklist, short reflection. |
| Phase 01 | Maps to team development, pull request review, and engineering onboarding. | Product, engineering, QA, security, operations, architecture | Development workflow evidence, feature map, review checklist.; Commit history, branch evidence, passing local checks, reviewer notes. |
| Phase 02 | Maps to QA practice, CI quality gates, and release approval evidence. | Product, engineering, QA, security, operations, architecture | Quality strategy, test checklist, screenshot evidence, quiz results.; Passing tests, expected output, accepted screenshots, defect notes. |
| Phase 03 | Maps to secure SDLC, security review, compliance preparation, and risk acceptance. | Product, engineering, QA, security, operations, architecture | Security review, threat notes, control checklist, remediation plan.; No secrets in repository, accepted risk notes, security screenshot evidence. |
| Phase 04 | Maps to AI governance, model-risk review, and responsible automation. | Product, engineering, QA, security, operations, architecture | Prompt records, reviewed outputs, limitation notes, validation checklist.; Prompt log, reviewer sign-off, corrected output, reflection. |
| Phase 05 | Maps to SRE, operations handoff, change management, and support readiness. | Product, engineering, QA, security, operations, architecture | Operations checklist, deployment evidence, monitoring notes, incident reflection.; Health check output, logs, dashboard screenshot, rollback notes. |
| Phase 06 | Maps to enterprise DevSecOps, compliance evidence, architecture review boards. | Product, engineering, QA, security, operations, architecture | Governance map, release checklist, control evidence, architecture review notes.; Pipeline evidence, approval trace, policy output, release notes. |
