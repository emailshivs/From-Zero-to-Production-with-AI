# Quiz Framework

## Purpose

Quizzes measure comprehension, applied judgment, and readiness to continue. They should reinforce learning rather than trick learners.

## Supported Question Types

The handbook supports:

- Multiple choice.
- Scenario based.
- Hands-on challenge.
- Reflection.
- Answer key.

## Multiple Choice Questions

Use multiple choice for terminology, cause and effect, command purpose, and concept comparison.

Rules:

- Provide one clearly correct answer unless marked as multi-select.
- Use plausible distractors.
- Avoid joke answers.
- Avoid “all of the above” unless pedagogically necessary.
- Explain why the correct answer is correct.

Format:

```markdown
### Question 1

What is the main purpose of validation?

A. Option A  
B. Option B  
C. Option C  
D. Option D  

**Answer:** B

**Explanation:** Brief explanation.
```

## Scenario-Based Questions

Use scenarios to test judgment in realistic engineering situations.

A good scenario includes:

- Context.
- Constraint.
- Decision point.
- Expected reasoning.

## Hands-On Challenges

Use hands-on challenges when learners must modify, run, inspect, or validate something.

Required elements:

- Task.
- Starting point.
- Constraints.
- Expected result.
- Validation method.

## Reflection Questions

Use reflection questions to connect the chapter to professional practice.

Good reflection prompts:

- What trade-off did this design introduce?
- How would this change in a production environment?
- What evidence would convince a reviewer that the work is correct?

## Answer Key

Every quiz must include an answer key.

Answer keys must include:

- Correct answer.
- Short explanation.
- Reference to relevant chapter section when useful.
- Rubric for hands-on or reflection questions.

## Difficulty Balance

A standard quiz should contain:

- 40% recall and terminology.
- 40% application and interpretation.
- 20% scenario judgment or reflection.

## Review Checklist

Before publishing a quiz, confirm that:

- Questions align with learning objectives.
- Answers are unambiguous.
- Distractors are plausible but not misleading.
- Scenario questions have enough context.
- Hands-on challenges include validation.
